# Stage 1 任务流协议（通用版 v1.0）

## 💡 核心定位

Stage 1 是全案多 Agent 协作流水线的“探路者与数字化建模机”**。
它的职责**不是翻译，而是将混沌、高密度的非结构化原文，转化为下游 Stage 2（翻译模型）可读、可控、防暴走的“逻辑坐标系”**与**“排雷图”。

---

## 一、 绝对红线：Stage 1 的“五不原则”

任何情况下，Stage 1 违反以下原则即视为“越权（Usurpation）”致命错误：

1. **不翻译**：严禁输出任何汉文翻译尝试（句法成分分析除外）。
2. **不解义**：严禁在 `notes` 中解释文本的深层宗派义理。
3. **不解码密义**：严禁提前解读密义符号（如明点、空色、婆伽等），保持其字面封装。
4. **不补背景**：严禁自行补充文本外的历史、人物或宗派背景。
5. **不判术语**：仅负责标记“术语命中”，绝不作最终的翻译决策。

---

## 二、 切分与拓扑原则（基于实战反哺的 3 大铁律）

### 铁律 1：全字符无损覆盖（Zero Character Loss）

* **痛点来源**：wzz-1 切片曾因引文闭合助词（`ཅེས་པ`）后接散文而发生致命漏句。
* **执行规范**：Stage 1 切分的所有节点（从 Node 1 到 Node N），其 `boundary_start` 与 `boundary_end` 所囊括的文本，拼接后必须 **100% 等于原始输入文本**。严禁在引文与散文的交界处、段落末尾丢失任何字符。

### 铁律 2：嵌套引文的扁平化切缝（Flat Slicing for Nested Quotes）

* **痛点来源**：wzz-1 中一个长散文句内部嵌套了两段引文，导致节点跨度极大，极易产生翻译漂移。
* **执行规范**：严禁深层嵌套。遇到散文包含引文时，必须执行“扁平切缝法”：
* `[G-x]` 前导散文（引出语）
* `[Q-y]` 引文本体
* `[G-z]` 总结或过渡散文



### 铁律 3：悬垂引导句的独立锚定（Dangling Guide Anchoring）

* **痛点来源**：wzz-2 中以 `ཏེ།`（即/因）、`ལྟར།`（如下）结尾的悬垂句被生硬切断，导致逻辑断层。
* **执行规范**：若句子本质是引发后文展开的引导语，不得强行并入前句或后句，应独立切分为节点，并在逻辑骨架中标注 `悬垂引出（ཏེ།） → 下游节点展开`。

## 四、 节点前缀与逻辑骨架（拓扑坐标系）

### 1. 节点前缀 (Prefix)

* **G (Gzhung)**：标准散文/义理解释句。
* **Q (Quote)**：经续、论典之引文。
* **R (Refutation)**：驳斥、判教、他见批判结构。
* **C (Core)**：核心定义句、性相宣示句。
* **T (Tantra/Tech)**：实修技术、步骤、气脉指令。
* **F (Final)**：段落总结/科判收束句。

### 2. 核心句法类型 (syntax_type)

从以下分类中选取最贴切者：
`因果链` / `条件句` / `让步句` / `枚举结构` / `是非判教句` / `双重否定` / `祈使句` / `复合句` / `对比判教句` / `悬垂引导句`。

### 3. 逻辑骨架公式 (syntax_components)

提取原文的逻辑主干，采用极简等式：

* `先破（མ་ཡིན་ཏེ།） = 非……`
* `后立（ཡིན་ནོ།） = 乃是……`
* `条件（ན） = 若……`
* `让步（ཀྱང་） = 虽……`

## 五、 高危风险雷达库（Risk Flags）

Stage 1 必须像扫雷器一样，在 `risk_flags` 字段中打上预警标签。下游 Stage 2 一旦看到这些标签，必须强制触发特定的防御逻辑。

**【必选标准标签库】**

1. `⚠️双重极性须完整保留`：针对“既不脱离世俗，亦非世俗即法身”等不即不离的宗义结构，严禁下游压扁。
2. `⚠️否定范围须限定`：针对“外境不成立（但显现存在）”，严禁下游将其译为彻底的虚无断灭。
3. `⚠️让步结构须完整保留`：针对带有 ཀྱང་ / ཡང་ 等字的逻辑转折。
4. `⚠️独立概念×N须各别译出`：针对密集枚举（如不可言、不可思、不可见），严禁下游合并为一句“不可思议”。
5. `⚠️是非判教句须完整保留`：严禁将作者犀利的判教改写为模棱两可的评论。
6. `⚠️引文密义不预先解码`：出现气脉、明点、佛智等词时，提醒下游字面直译。
7. `⚠️主语指代须确认`：针对藏文频繁省略主语或 དེ (彼) 指代不明的区域。
8. `⚠️悬垂断层防丢`：针对连续句被切断的交界处。
9. `⚠️术语表命中须核查`：提醒下游调用并死守术语表。

## 六、 异常拦截与主动报告机制（unresolved 字段）

Stage 1 具备“学术质疑权”。当遇到以下情况时，必须在 JSON 的 `unresolved` 字段中报警，不得强行掩盖：

1. **疑似错字/衍文**：发现疑似受前文模板干扰残留的脏数据。
2. **引文边界不明**：藏文无清晰的 `ཅེས་གསུངས` (如是说) 闭合时，指明边界争议。
3. **科判层级错乱**：原文的数字编号（如“四项”与“十六种”）在语法上存在冲突时。
4. **校勘版本争议**：原文自身提到“旧译为XX，应改为XX”时，提示下游特殊处理。
